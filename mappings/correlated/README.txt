В двух файлах находится один и тот же маппинг для разных случаев:

1) correlated_template.json - для паппита, по примеру из "modules_iss_files_elasticsearch_templates_cef.json",
для reindex от kibana tools

2) correlated.json - чтобы вручную поменять маппинг индекса при создании клона через kaizen.



Как поменять индекс уже созданного индекса?
1) [SSH] Останавливаем службу EA.
2) [Kaizen] Создаем клон индекса correlated с новым именем correlated_clone
3) [Kaizen] Удаляем оригинальный индекс correlated
4) [Kaizen] Создаем клон индекса correlated_clone:
    4.1) Название: correlated
    4.2) Копируем содержимое correlated.json и вставляем в окно маппинга создаваемого индекса
5*) [Kaizen] Буфферный индекс correlated_clone пока моно оставить, на всякий случай.
    Если прям надо удалить клон, то удалите через 30 дней нормальной работы EA с новым индексом.
6) [SSH] Запускаем службу EA
7*) [SSH] Проверяем логи EA, нет ли ошибок.

* - дополнительные пункты
